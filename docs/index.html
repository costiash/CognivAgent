Built for transcript-heavy research, knowledge extraction, and agent-driven exploration.

CognivAgent is a FastAPI application that transcribes videos, extracts entities and relationships, and turns the result into an interactive knowledge graph you can inspect, export, and query. It combines the Claude Agent SDK, OpenAI transcription, and a modular frontend designed for rich graph workflows.

What it is

A modular AI system for transcript-first knowledge work.

  • BackendFastAPI + Uvicorn
  • AI stackClaude Agent SDK + OpenAI
  • Graph layerNetworkX + Cytoscape.js
  • FrontendVanilla ES modules

The project is ideal for researchers, builders, and analysts who need more than a transcript viewer. It preserves source citations, shows extraction progress, and keeps graph exploration close to the original evidence.

Current status

Research-oriented and rapidly evolving.

CognivAgent is not presented as production-ready infrastructure. The current focus is fast iteration on transcription, extraction quality, developer tooling, and async job workflows.

Core capabilities

From transcript ingestion to graph export.

The product surface is intentionally broad: transcription, knowledge extraction, entity resolution, graph visualization, transcript search, audit trails, and asynchronous job orchestration all live in one cohesive workflow.

Video transcription with domain context

Process local videos and supported YouTube transcript flows with gpt-4o-transcribe, then keep transcript text close to the downstream extraction pipeline.

Knowledge graph extraction

Bootstrap domain schemas, extract entities and relationships, and keep source citations so every graph node can be traced back to the transcript evidence.

Interactive graph exploration

Explore results in a Cytoscape.js interface with search, node inspection, filtering, and export paths for external graph tooling.

Workflow

Designed for repeatable research loops.

The flow keeps discovery and structure in sync: start with a video, confirm the emerging schema, then use it to process additional material without losing provenance.

Knowledge graph lifecycle

  1. Create a project around a topic or investigation.
  2. Bootstrap a schema from the first transcript and confirm discoveries.
  3. Extract entities and relationships from additional videos.
  4. Inspect, refine, search, and export the resulting graph.
Architecture snapshot

Queue-based SessionActor pattern

HTTP requests enqueue work to a dedicated session actor, which keeps Claude SDK access serialized and avoids concurrent cancel-scope failures while still supporting activity streaming and background job completion callbacks. 

HTTP request → input_queue → SessionActor → response_queue → UI
Why teams notice it

It treats transcripts like structured evidence.

Instead of stopping at text output, CognivAgent links transcripts, extracted entities, graph relationships, and UI inspection into one research surface built for synthesis.

Quick start

Run the project locally in minutes.

The default development path uses uv, plus Anthropic and OpenAI API keys. Docker is also available for containerized runs.

Installation 
git clone https://github.com/costiash/CognivAgent.git
cd CognivAgent
uv sync
cp .env.example .env
# add ANTHROPIC_API_KEY and OPENAI_API_KEY
uv run python -m app.main
Validation 
uv run pytest
uv run mypy .
uv run ruff check .
uv run ruff format --check .
Important note

Built for localhost and research workflows.

The repository explicitly warns that production deployment needs additional security hardening. For now, treat this as a powerful local-first tool for exploration, prototyping, and contributor collaboration.
